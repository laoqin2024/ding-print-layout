# 钉钉审批套印系统操作指导手册（中文）

## 1. 项目目标

本系统用于在钉钉 H5 场景下，读取审批实例信息并将签名/附件按坐标套印到 PDF 模板，供预览与打印。

---

## 2. 运行前准备

### 2.1 Python 侧

1. 进入项目根目录：`/root/dingtalk-h5-app`
2. 激活虚拟环境（如使用内置）：`source dingstream/bin/activate`
3. 设置环境变量（建议使用 `.env`）

### 2.2 前端构建侧

1. 进入 `frontend/`
2. 安装依赖：`npm install`
3. 生产构建：`npm run build`

构建产物会输出到：`static/dist`

---

## 3. 配置文件说明

### 3.1 环境变量文件

参考模板：`.env.example`

必填变量：

- `DINGTALK_APP_KEY`
- `DINGTALK_APP_SECRET`
- `DINGTALK_CORP_ID`
- `FLASK_SECRET_KEY`

系统启动时会校验上述变量，不完整将直接报错并拒绝启动（安全策略）。

### 3.2 业务配置文件

- 文件路径：`data/process_configs.json`
- 作用：维护 `ProcessCode -> 模板 -> 坐标 zones` 映射

---

## 4. 页面操作流程

### 4.1 门户页（移动端）

- 页面：`/`
- 行为：钉钉鉴权后加载“我的审批单”列表
- 可点击“确认预览”进入打印预览页

### 4.2 管理后台（PC）

- 页面：`/admin`
- 可上传 PDF 底图、管理流程配置
- “编辑坐标”进入可视化测绘页面

### 4.3 可视化测绘

- 页面：`/admin/edit`
- 支持拖拽创建区域、缩放、JSON 实时同步
- 保存后写入 `data/process_configs.json`

### 4.4 打印预览

- 页面：`/print_view`
- 使用 PDF.js 渲染，支持“立即打印”

---

## 5. 常用命令

### 5.1 启动后端

- `python3 run.py`

### 5.2 前端命令（`frontend/` 目录）

- 开发：`npm run dev`
- 构建：`npm run build`
- 预览：`npm run preview`
- Lint：`npm run lint`
- 格式化：`npm run format`

---

## 6. 故障排查

1. 启动报 `Missing required environment variables`
   - 检查 `.env` 是否完整，并确认 shell 已加载。
2. 页面样式异常
   - 确认已执行 `npm run build` 且 `static/dist` 存在。
3. 预览页 PDF 空白
   - 检查审批实例是否含可下载签名/附件 URL；
   - 检查 `data/process_configs.json` 中坐标和模板是否匹配。
