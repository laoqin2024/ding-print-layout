## 技术细节

### 路径处理策略

所有脚本现在使用以下模式自动检测项目根目录：

**Shell 脚本：**
```bash
SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
```

**Python 脚本：**
```python
SCRIPT_DIR = Path(__file__).parent.absolute()
sys.path.insert(0, str(SCRIPT_DIR))
```

### 兼容性改进

1. **Python 解释器**：使用系统 `python3` 而非硬编码路径
2. **日志文件**：从 `/tmp` 移到项目目录，便于管理
3. **URL 配置**：从远程 IP 改为 `localhost`
4. **macOS 兼容**：修复 `stat` 命令语法差异

## 注意事项

### 1. Python 环境

确保已安装所需依赖：
```bash
pip3 install -r requirements.txt
```

### 2. 环境变量

检查 `.env` 文件中的钉钉凭证是否正确：
- `DINGTALK_APP_KEY`
- `DINGTALK_APP_SECRET`
- `DINGTALK_CORP_ID`

### 3. 文件权限

所有脚本已设置可执行权限。如需重新设置：
```bash
chmod +x *.sh
```

### 4. 端口占用

如果 5000 端口被占用，修改 `run.py` 中的端口号：
```python
app.run(host="0.0.0.0", port=5000, debug=True)
```
