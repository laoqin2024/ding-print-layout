修复时间: 2026-05-04 18:10
问题: 控件内文字有横向居中但是没有纵向居中

================================================================================
🐛 问题分析
================================================================================

原始代码:
```python
y = rect.y0 + fontsize + i * line_step
```

问题:
- 文本从矩形顶部开始绘制
- 没有考虑纵向居中
- 导致文字总是靠近顶部

================================================================================
✅ 解决方案
================================================================================

修改后的代码:
```python
# 计算纵向居中偏移
total_text_height = len(lines) * line_step
vertical_offset = max(0, (rect.height - total_text_height) / 2.0)

y = rect.y0 + fontsize + vertical_offset + i * line_step
```

工作原理:
1. 计算所有文本行的总高度: `total_text_height = len(lines) * line_step`
2. 计算纵向偏移量: `vertical_offset = (rect.height - total_text_height) / 2.0`
3. 将偏移量添加到 Y 坐标: `y = rect.y0 + fontsize + vertical_offset + i * line_step`

示例:
- 矩形高度: 40 pt
- 文本行数: 1
- 行高: 12 pt (fontsize 10 * line_height 1.2)
- 总文本高度: 12 pt
- 纵向偏移: (40 - 12) / 2 = 14 pt
- 文本将在矩形中垂直居中

================================================================================
⚠️ 注意事项
================================================================================

1. 多行文本
   - 多行文本会作为一个整体垂直居中
   - 每行之间的间距由 line_height 控制

2. 文本溢出
   - 如果文本高度超过控件高度
   - vertical_offset 会是 0（不会是负数）
   - 文本会从顶部开始绘制

3. 字体大小
   - 纵向居中考虑了字体大小和行高
   - 确保 line_height 设置合理（通常 1.2-1.5）

```python
def _draw_text_with_spacing(
    # ... 现有参数 ...
    vertical_align: str = "middle",  # top, middle, bottom
):
    # ...
    
    if vertical_align == "top":
        vertical_offset = 0
    elif vertical_align == "bottom":
        vertical_offset = max(0, rect.height - total_text_height)
    else:  # middle
        vertical_offset = max(0, (rect.height - total_text_height) / 2.0)
    
    y = rect.y0 + fontsize + vertical_offset + i * line_step
```

- 顶部对齐
- 居中对齐（默认）
- 底部对齐
