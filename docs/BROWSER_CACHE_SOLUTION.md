================================================================================
🔧 浏览器缓存问题解决方案
================================================================================

问题: 修改了 HTML 文件，但浏览器显示的还是旧版本

原因: 浏览器缓存了旧的 HTML 文件

================================================================================
✅ 解决方法
================================================================================

### 方法 1: 强制刷新（最简单，推荐）

**Windows/Linux**: 
```
Ctrl + Shift + R
```

**Mac**: 
```
Cmd + Shift + R
```

这会强制浏览器重新加载所有资源，包括 HTML、CSS、JavaScript

---

### 方法 2: 清空缓存并硬性重新加载

**Chrome**:
1. 按 `F12` 打开开发者工具
2. 右键点击地址栏旁边的刷新按钮
3. 选择"清空缓存并硬性重新加载"

**Firefox**:
1. 按 `Ctrl+Shift+Delete` (Windows) 或 `Cmd+Shift+Delete` (Mac)
2. 选择"缓存"
3. 点击"立即清除"
4. 刷新页面

**Safari**:
1. 按 `Cmd+Option+E` 清空缓存
2. 刷新页面

---

### 方法 3: 使用隐私/无痕模式

打开一个新的隐私/无痕窗口，访问页面。

这样可以确保没有缓存影响。

**Chrome**: `Ctrl+Shift+N` (Windows) 或 `Cmd+Shift+N` (Mac)
**Firefox**: `Ctrl+Shift+P` (Windows) 或 `Cmd+Shift+P` (Mac)
**Safari**: `Cmd+Shift+N`

---

### 方法 4: 重启服务器（如果上面方法无效）

```bash
cd "/Volumes/MyDisk/App programs/dingtalk-h5-app"
pkill -f "python.*run.py"
python run.py
```

然后再强制刷新浏览器（Ctrl+Shift+R 或 Cmd+Shift+R）

================================================================================
🔍 如果还是看不到，请检查
================================================================================

### 1. 检查浏览器控制台是否有错误

1. 按 `F12` 打开开发者工具
2. 点击 `Console` 标签页
3. 查看是否有红色的错误信息
4. 如果有错误，请把错误信息发给我

---

### 2. 检查网络请求

1. 在开发者工具中点击 `Network` 标签页
2. 刷新页面（Ctrl+R 或 Cmd+R）
3. 在列表中找到 `designer_edit` 或 `edit` 请求
4. 点击该请求，查看 `Response` 标签页
5. 搜索 "vertical_align" 或 "垂直对齐"
6. 如果找到了，说明服务器返回的是新版本
7. 如果找不到，说明服务器还在返回旧版本（需要重启服务器）

---

### 3. 检查 HTML 元素

1. 在开发者工具中点击 `Elements` 标签页
2. 按 `Ctrl+F` (Windows) 或 `Cmd+F` (Mac) 打开搜索
3. 搜索 "prop-vertical-align"
4. 如果找到了，说明 HTML 已经更新
5. 如果找不到，说明浏览器还在使用缓存

---

### 4. 检查文件修改时间

```bash
ls -la templates/designer_edit.html
```

查看文件的修改时间，确认是最近修改的。

📝 常见问题
================================================================================

### Q: 为什么强制刷新还是看不到？

A: 可能的原因：
1. 浏览器缓存非常顽固，需要清空缓存
2. 服务器没有重启，还在使用旧的模板
3. 浏览器扩展（如广告拦截器）干扰了加载

解决方法：
1. 使用隐私/无痕模式访问
2. 重启服务器
3. 禁用浏览器扩展

---

### Q: 我看到了垂直对齐选择器，但选择后没有效果？

A: 可能的原因：
1. JavaScript 文件也被缓存了
2. 需要同时刷新 JavaScript 文件

解决方法：
1. 强制刷新（Ctrl+Shift+R）
2. 清空缓存
3. 重启服务器

---

### Q: 我在隐私模式下看到了，但正常模式下看不到？

A: 这说明是缓存问题。

解决方法：
1. 在正常模式下清空缓存
2. 或者继续使用隐私模式
