# 附件底图画布预览功能

## 更新时间
2026-05-02

## 功能说明

现在可以在设计器画布中实时预览附件底图了！当勾选"附件底图配置"时，系统会自动从钉钉实例中获取附件并显示在画布上。

## 使用方法

### 步骤1：填写必要信息

1. **流程代码（process_code）**
   - 填写或选择要设计的流程

2. **实例 ID**
   - 填写一个包含附件的真实实例 ID
   - 这个实例必须包含你要使用的附件字段

### 步骤2：启用附件底图配置

1. 找到"附件底图配置"区域
2. ✅ 勾选"启用"复选框
3. 系统会自动显示配置选项

### 步骤3：配置附件参数

1. **选择附件字段**
   - 从下拉框选择包含 PDF 附件的字段
   - 如："附件"、"图纸"、"DDAattachment"等

2. **设置附件索引**
   - 如果字段有多个附件，指定使用第几个
   - 索引从 0 开始（0 = 第一个附件）

3. **设置 PDF 页码**
   - 如果附件 PDF 有多页，指定使用第几页
   - 页码从 0 开始（0 = 第一页）

### 步骤4：查看画布预览

配置完成后，系统会：
1. 自动从钉钉实例获取附件
2. 下载并解析 PDF 文件
3. 渲染指定页面为图片
4. 显示在设计器画布背景上

**画布标题会显示状态：**
- `正在加载附件底图...` - 正在下载和处理
- `画布（A4 横向）- 附件底图已加载` - 加载成功
- `附件底图加载失败：错误信息` - 加载失败

## 功能特点

### 实时预览
- ✅ 切换附件字段时自动刷新
- ✅ 切换附件索引时自动刷新
- ✅ 切换页码时自动刷新
- ✅ 取消勾选时恢复正常底图

### 智能提示
- 画布标题实时显示加载状态
- 加载失败时弹出详细错误信息
- 提示用户检查配置是否正确

### 性能优化
- 使用缓存机制，避免重复下载
- 异步加载，不阻塞界面操作
- 自动释放内存（Blob URL）

## 使用场景

### 场景1：设计新布局
```
1. 找一个包含附件的实例
2. 启用附件底图配置
3. 选择附件字段和页码
4. 在画布上看到真实的附件底图
5. 直接在附件上布局控件
6. 位置精确，无需反复预览 PDF
```

### 场景2：调整现有布局
```
1. 打开已有的布局配置
2. 启用附件底图配置
3. 查看控件在真实附件上的位置
4. 微调控件位置和大小
5. 保存布局
```

### 场景3：多页附件处理
```
1. 附件 PDF 有多页
2. 设置页码为 0，查看第一页
3. 布局第一页的控件
4. 设置页码为 1，查看第二页
5. 布局第二页的控件
```

## 错误处理

### 错误1：实例 ID 为空
**提示**：`请填写流程代码、实例ID和选择附件字段`

**解决**：填写实例 ID

### 错误2：附件字段不存在
**提示**：`Field xxx not found`

**解决**：
1. 检查实例是否包含该字段
2. 尝试重新拉取表单控件
3. 选择正确的附件字段

### 错误3：附件索引超出范围
**提示**：`Attachment index X out of range`

**解决**：
1. 检查该字段有几个附件
2. 索引从 0 开始
3. 如果只有 1 个附件，索引应该是 0

### 错误4：页码超出范围
**提示**：`Page index X out of range (total Y pages)`

**解决**：
1. 附件 PDF 只有 Y 页
2. 页码从 0 开始
3. 最大页码是 Y-1

### 错误5：附件不是 PDF
**提示**：`Invalid attachment format` 或其他错误

**解决**：
1. 确认附件是 PDF 文件
2. 不支持图片、Word 等其他格式
3. 选择正确的附件字段

## 注意事项

### 1. 实例 ID 必须真实存在
- 不能使用假的或过期的实例 ID
- 实例必须包含配置的附件字段

### 2. 附件必须是 PDF 格式
- 不支持图片（JPG、PNG）
- 不支持 Word、Excel 等文档
- 如果需要使用图片，请先转换为 PDF

### 3. 网络和性能
- 首次加载需要下载附件，可能较慢
- 大文件（>10MB）可能需要等待
- 建议使用较小的 PDF 文件作为底图

### 4. 缓存机制
- 浏览器会缓存已加载的图片
- 切换参数时会重新加载
- 刷新页面会清除缓存

### 5. 权限要求
- 需要有访问钉钉实例的权限
- 需要有下载附件的权限
- 如果权限不足，会返回错误
