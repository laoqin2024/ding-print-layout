### 1. 根据方向自动选择空白模板
```python
# 如果使用附件底图但没有 base_pdf，根据方向使用对应的空白模板
if using_attachment_bg and not base_pdf:
    if orientation == "p":
        base_pdf = "blank_portrait.pdf"  # 纵向空白模板
    else:
        base_pdf = "blank_landscape.pdf"  # 横向空白模板
```

### 2. 简化路径处理
```python
# 所有模板都在 templates 目录下，统一处理
pdf_path = cfg.pdf_template_dir / base_pdf

    return jsonify({"ok": False, "msg": f"底稿不存在: {base_pdf}"}), 404
```

📊 使用场景
================================================================================

### 场景 1: 附件底图模式（自动）
```
用户操作:
- 启用附件底图
- 选择横向布局
- 不选择 base_pdf

系统行为:
→ 自动使用 blank_landscape.pdf
→ 附件覆盖在空白模板上
→ 控件绘制在附件上
```

### 场景 2: 附件底图模式（手动）
```
用户操作:
- 启用附件底图
- 选择纵向布局
- 手动选择 blank_portrait.pdf

系统行为:
→ 使用选择的空白模板
→ 附件覆盖在空白模板上
→ 控件绘制在附件上
```

### 场景 3: 普通模板模式
```
用户操作:
- 禁用附件底图
- 勾选"使用模板"
- 手动选择 blank_landscape.pdf

系统行为:
→ 使用空白模板作为背景
→ 控件直接绘制在空白页上
→ 适合纯控件布局
```

### 场景 4: 不使用模板
```
用户操作:
- 禁用附件底图
- 不勾选"使用模板"

系统行为:
→ 创建空白 PDF
→ 控件直接绘制
→ 最简单的模式
```
