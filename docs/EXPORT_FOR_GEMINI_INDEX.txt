🎯 KEY INFORMATION FOR GEMINI
================================================================================

1. **DingTalk Print API**
   - Uses dd.biz.util.print() for native printing
   - Requires dd.config with jsApiList: ['biz.util.print']
   - Falls back to window.print() if unavailable
   - Implementation: EXPORT_FOR_GEMINI_01_frontend_print.js.txt (lines 120-140)

2. **Two Print Systems**
   - Legacy: process_configs.json + pdf_service.py overlay
   - Designer: print_layouts.json + PyMuPDF direct rendering
   - Routing logic: EXPORT_FOR_GEMINI_02_backend_printing_routes.py.txt (lines 70-100)

3. **PDF Generation**
   - Backend: PyMuPDF (fitz) for PDF manipulation
   - Frontend: PDF.js for browser rendering
   - Service: EXPORT_FOR_GEMINI_03_pdf_service.py.txt (971 lines)

4. **Attachment Background Feature**
   - DingTalk returns fileId, not direct URL
   - Must call API to convert fileId → download URL
   - Recently fixed (2026-05-04)
   - Details: EXPORT_FOR_GEMINI_00_PROJECT_CONTEXT.txt (lines 150-180)

5. **Templates**
   - Legacy: EXPORT_FOR_GEMINI_04_template_print.html.txt
   - Designer: EXPORT_FOR_GEMINI_05_template_designer_print.html.txt
   - Both use Jinja2 templating

🔧 TECHNICAL STACK
================================================================================

Backend:
- Flask (Python web framework)
- PyMuPDF (fitz) - PDF manipulation
- Requests - HTTP client for DingTalk API

Frontend:
- Vanilla JavaScript (ES6+)
- PDF.js - PDF rendering in browser
- Tailwind CSS - Styling
- DingTalk JSAPI - Native mobile features

DingTalk Integration:
- Workflow API - Get approval instances
- File API - Download attachments
- JSAPI - Native print function
