🐛 问题分析
================================================================================

问题:
- designer_print.html 使用 window.print()
- window.print() 会打印整个 HTML 页面
- 而不是打印 PDF 文件内容

期望行为:
- 在钉钉中：使用 dd.biz.util.print() 打印 PDF
- 在浏览器中：使用 window.print() 打印页面（PDF 会被打印）

✅ 修复方案
================================================================================

修改文件: templates/designer_print.html

修改 1: 保存 PDF URL
---------------------------------------------------------------
```javascript
const blob = await res.blob();
const url = URL.createObjectURL(blob);

// 保存 PDF URL 供打印使用
window.currentPdfUrl = url;  // ← 新增
```

修改 2: 增强打印按钮逻辑
---------------------------------------------------------------
修改前:
```javascript
printFab?.addEventListener('click', () => {
  window.print();  // ← 总是打印整个页面
});
```

修改后:
```javascript
printFab?.addEventListener('click', () => {
  // 尝试使用钉钉原生打印
  if (window.dd && dd.env && dd.env.platform !== "notInDingTalk") {
    const pdfUrl = window.currentPdfUrl;
    if (pdfUrl) {
      const fullUrl = pdfUrl.startsWith('http') ? pdfUrl : window.location.origin + pdfUrl;
      dd.ready(function () {
        try {
          dd.biz.util.print({
            url: fullUrl,
            onSuccess: function () {
              console.log('[DingTalk] 正在唤起钉钉打印...');
            },
            onFail: function (err) {
              console.error('[DingTalk] 钉钉打印失败:', err);
              window.print();
            },
          });
        } catch (e) {
          console.error('[DingTalk] 钉钉打印异常:', e);
          window.print();
        }
      });
    } else {
      window.print();
    }
  } else {
    window.print();
  }
});
```

⚠️ 注意事项
================================================================================

1. PDF URL 必须是完整的 URL
   - blob: URL 需要转换为绝对路径
   - 或者使用服务器上的 PDF 文件路径

2. 钉钉打印需要 JSAPI 配置
   - 确保 dd.config 已正确配置
   - 确保 jsApiList 包含 "biz.util.print"

3. 降级逻辑
   - 如果钉钉打印失败，会降级到 window.print()
   - 如果不在钉钉环境，直接使用 window.print()
