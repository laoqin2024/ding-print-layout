### 权限控制流程
```
用户登录钉钉
    ↓
访问打印门户
    ↓
后端获取用户信息（userid, name, dept_name）
    ↓
加载流程配置（print_layouts.json）
    ↓
检查每个流程的权限配置
    ├─ 无配置 → 允许访问
    ├─ 有配置 → 检查部门和用户
    │   ├─ 部门匹配 ✓
    │   ├─ 用户匹配 ✓
    │   └─ 都不匹配 ✗
    └─ 返回有权限的流程列表
        ↓
前端显示流程筛选器和审批单列表
```

## 🔧 使用指南

### 快速开始

1. **查看当前配置**
   ```bash
   cd /root/dingtalk-h5-app
   python3 verify_permissions.py
   ```

2. **为流程添加权限**
   - 编辑 `data/print_layouts.json`
   - 在流程配置中添加 `permissions` 字段
   - 参考 `data/PERMISSION_SETUP_GUIDE.md`

3. **验证配置**
   ```bash
   python3 verify_permissions.py
   ```

4. **测试**
   - 使用不同部门用户登录钉钉
   - 访问打印门户验证权限

### 配置示例

```json
{
  "PROC-XXXXX": {
    "name": "设计图纸变更通知单",
    "permissions": {
      "allowed_depts": ["技术中心一部", "品保", "总经办"]
    },
    "base_pdf": "template.pdf",
    "items": [...]
  }
}
```

## 🔐 安全性说明

1. **权限检查在后端执行**
   - 前端只负责显示，不做权限判断
   - 后端 API 检查用户权限后返回数据

2. **用户信息来源**
   - 通过钉钉 authCode 获取 userid
   - 从 `users.json` 加载用户详细信息
   - 确保用户信息的准确性

3. **默认安全策略**
   - 如果用户信息加载失败，权限检查失败
   - 如果配置错误，倾向于拒绝访问
   - 未配置权限的流程保持开放（向后兼容）
