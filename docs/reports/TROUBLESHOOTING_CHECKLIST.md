## 问题1：部门控件不会换行

### 快速排查步骤

#### 步骤1：强制刷新浏览器
```
Windows/Linux: Ctrl + Shift + R
Mac: Cmd + Shift + R
```

#### 步骤2：运行调试脚本
1. 打开设计器页面
2. 按 F12 打开开发者工具
3. 切换到 Console 标签
4. 访问：`http://your-server/static/debug_designer.js`
5. 复制脚本内容到控制台运行
6. 查看输出信息

#### 步骤3：检查操作流程
- [ ] 选中了部门控件（边框变粉色）
- [ ] 勾选了"文本自动换行"复选框
- [ ] **点击了"更新选中项"按钮** ⚠️ 最重要！
- [ ] 点击了"保存布局"按钮

#### 步骤4：手动测试
在浏览器控制台输入：
```javascript
// 1. 查看当前选中的控件
const idx = Array.from(state.selected)[0];
console.log('text_wrap:', state.items[idx].text_wrap);

// 2. 手动启用换行
state.items[idx].text_wrap = true;
renderItems();

// 3. 查看 DOM 元素样式
const el = document.querySelector(`[data-idx="${idx}"]`);
console.log('white-space:', window.getComputedStyle(el).whiteSpace);
```

### 可能的原因和解决方案

#### 原因1：没有点击"更新选中项"
**症状**：勾选了复选框，但控件没有变化

**解决**：
1. 勾选"文本自动换行"
2. **必须点击"更新选中项"按钮**
3. 观察画布变化

#### 原因2：浏览器缓存
**症状**：刷新页面后还是旧版本

**解决**：
1. 按 Ctrl+Shift+R 强制刷新
2. 或清除浏览器缓存
3. 或在隐私/无痕模式下打开

#### 原因3：CSS 样式未生效
**症状**：JS 配置正确，但样式不对

**检查**：
在开发者工具中检查 `.designer-item` 的样式是否包含：
```css
box-sizing: border-box;
max-width: 100%;
word-wrap: break-word;
```

**解决**：
如果没有这些样式，说明 HTML 文件没有更新。需要重启服务器。

#### 原因4：控件宽度太大
**症状**：文本没有超出宽度，所以看不到换行

**测试**：
1. 将控件宽度设置为 80px
2. 将字号设置为 14px
3. 输入长文本："集团质量管理部一部"
4. 应该能看到换行

## 问题2：附件底图不能预览

### 重要说明

**这是设计上的限制，不是 Bug！**

附件底图**无法在设计器画布中预览**，原因：
1. 附件来自钉钉实例，需要实时获取
2. 设计器画布是静态预览，追求快速响应
3. 实时获取附件会严重影响性能

### 正确的使用方式

#### 方式1：使用底稿 PDF（推荐用于设计阶段）
1. 底图来源选择：`base`（底稿）
2. 选择一个示例 PDF 文件
3. 在画布上布局控件
4. 画布可以实时预览底图

#### 方式2：使用附件底图（用于实际打印）
1. 先用方式1完成布局设计
2. 布局完成后，切换底图来源为：`attachment`（流程附件）
3. 配置附件字段、索引等
4. **点击"预览 PDF"按钮**查看实际效果
5. 画布会显示提示：`附件底图模式：请点击"预览 PDF"查看实际效果`

### 附件底图配置步骤

1. **启用附件底图**
   - 勾选"启用"复选框

2. **选择附件字段**
   - 从下拉框选择包含 PDF 附件的字段
   - 如："附件"、"图纸"、"DDAattachment"等

3. **设置附件索引**
   - 如果字段有多个附件，指定使用第几个
   - 索引从 0 开始（0 = 第一个附件）

4. **设置 PDF 页码**
   - 如果附件 PDF 有多页，指定使用第几页
   - 页码从 0 开始（0 = 第一页）

5. **选择应用模式**
   - `仅打印时生效`（推荐）：设计器预览快，实际打印用附件
   - `预览和打印都生效`：预览也用附件（慢但真实）

6. **预览效果**
   - 填写实例 ID
   - 点击"预览 PDF"按钮
   - 查看生成的 PDF 是否包含附件底图

### 附件底图不显示的原因

#### 原因1：实例 ID 为空
**解决**：填写一个真实的实例 ID

#### 原因2：附件字段选择错误
**解决**：确认选择的字段确实包含 PDF 附件

#### 原因3：附件索引错误
**解决**：检查该字段有几个附件，索引是否正确

#### 原因4：应用模式设置错误
**解决**：
- 如果是"仅打印时生效"，设计器预览不会显示附件
- 改为"预览和打印都生效"可以在预览中看到

## 临时解决方案

### 如果换行功能完全不工作

在浏览器控制台手动执行：

```javascript
// 1. 选中控件
const idx = Array.from(state.selected)[0];

// 2. 启用换行
state.items[idx].text_wrap = true;

// 3. 手动设置样式
const el = document.querySelector(`[data-idx="${idx}"]`);
el.style.whiteSpace = "pre-wrap";
el.style.wordBreak = "break-all";
el.style.overflowWrap = "break-word";
el.style.overflow = "hidden";

// 4. 保存配置
// 点击"保存布局"按钮
```
