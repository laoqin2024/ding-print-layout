# 变更日志 (CHANGELOG)

## [2.0.0] - 2026-05-02

### 新增功能 ✨

#### 流程模板筛选
- 在打印门户工具栏添加流程模板下拉选择器
- 支持按流程模板快速过滤审批单
- 流程列表按中文名称自动排序
- 状态栏显示当前筛选的流程名称和数量

#### 权限控制系统
- 支持基于部门的流程访问权限控制
- 支持基于用户ID的流程访问权限控制
- 支持部门和用户组合权限配置
- 权限配置立即生效，无需重启服务
- 向后兼容，未配置权限的流程保持对所有用户开放

#### 用户信息显示
- 页面顶部显示当前登录用户的姓名和部门
- 帮助用户确认身份和理解权限范围

### 改进优化 🎨

#### UI/UX 优化
- 优化工具栏布局，增加清晰的标签说明
- 流程模板筛选器独立成行，更加醒目
- 审批状态筛选器添加"审批状态筛选"标签
- 搜索框提示文本更新为"搜索标题/编号"
- 改进页面头部布局，信息层次更清晰

#### 后端优化
- 重构权限检查逻辑，提取为独立函数
- 添加用户信息加载函数
- API 返回更丰富的数据（流程模板列表、用户信息）
- 改进错误处理和边界情况处理

#### 前端优化
- 重构筛选逻辑，支持多维度组合筛选
- 优化状态显示，提供更多上下文信息
- 改进代码结构，提高可维护性

### 兼容性 🔄

- ✅ **向后兼容**: 未配置权限的流程保持原有行为
- ✅ **渐进增强**: 可以逐步为流程添加权限配置
- ✅ **优雅降级**: 配置错误时倾向于拒绝访问（安全优先）

### 安全性 🔐

- ✅ 权限检查在后端执行，前端不可绕过
- ✅ 用户信息通过钉钉 authCode 验证
- ✅ 默认安全策略：信息加载失败时拒绝访问
- ✅ 配置错误时倾向于拒绝访问

### 性能 ⚡

- ✅ 权限检查在数据加载时一次性完成
- ✅ 前端筛选操作在客户端执行，响应快速
- ✅ 配置文件缓存在内存中，避免重复读取

### 已知限制 ⚠️

1. 权限只控制打印门户的流程列表显示，不影响直接访问打印页面的URL
2. 权限配置依赖 `users.json` 的准确性，需要定期同步用户信息
3. 部门名称和用户ID必须完全匹配，区分大小写

### 迁移指南 📖

#### 从旧版本升级

1. **备份配置文件**
   ```bash
   cp data/print_layouts.json data/print_layouts.json.backup
   ```

2. **更新代码**
   - 拉取最新代码
   - 重新构建前端（已完成）

3. **（可选）配置权限**
   - 参考 `data/PERMISSION_SETUP_GUIDE.md`
   - 使用 `verify_permissions.py` 验证配置

4. **测试**
   - 访问打印门户验证新功能
   - 使用不同用户测试权限控制

#### 回滚方案

如需回滚到旧版本：
1. 恢复备份的配置文件
2. 回退代码到上一个版本
3. 重新构建前端
