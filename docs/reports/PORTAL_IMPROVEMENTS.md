## 优化内容

### 1. 流程模板筛选功能 ✅

#### 功能描述
- 在打印门户工具栏中新增"流程模板筛选"下拉框
- 用户可以按流程模板快速过滤审批单
- 下拉框选项按流程名称中文排序
- 支持"全部流程模板"选项查看所有审批单

#### 使用方式
1. 打开钉钉打印门户
2. 在工具栏中找到"流程模板筛选"下拉框
3. 选择要查看的流程模板
4. 列表自动更新，只显示该流程的审批单

#### 技术实现
- 前端：新增 `templateSelect` 元素和 `templateFilter` 状态
- 后端：API 返回 `templates` 字段，包含所有可用流程模板
- 筛选逻辑：在 `applyFilters()` 函数中按 `p_code` 过滤

### 2. 权限控制功能 ✅

#### 功能描述
- 支持基于部门和用户的流程访问权限控制
- 用户只能看到有权限访问的流程审批单
- 权限配置灵活，支持多种组合方式
- 配置立即生效，无需重启服务

#### 权限配置方式

在 `data/print_layouts.json` 中为流程添加 `permissions` 字段：

```json
{
  "PROC-XXXXX": {
    "name": "流程名称",
    "permissions": {
      "allowed_depts": ["部门A", "部门B"],
      "allowed_users": ["userid1", "userid2"]
    },
    "base_pdf": "...",
    "items": [...]
  }
}
```

#### 权限规则

| 配置方式 | 说明 | 示例 |
|---------|------|------|
| 无配置 | 所有用户可见 | 不添加 `permissions` 字段 |
| 按部门 | 只有指定部门用户可见 | `"allowed_depts": ["采购部"]` |
| 按用户 | 只有指定用户可见 | `"allowed_users": ["userid1"]` |
| 组合 | 必须同时满足部门和用户条件 | 同时配置两个字段 |

#### 技术实现
- 后端：`_check_process_permission()` 函数检查权限
- 后端：`_load_user_info()` 函数从 `users.json` 加载用户信息
- 前端：显示用户姓名和部门信息
- API：返回 `user_info` 字段，包含当前用户信息

## 使用指南

### 配置流程权限

1. **获取部门名称和用户ID**
   - 访问 `/admin/users` 人员管理页面
   - 查看用户的部门名称和 UserID

2. **编辑配置文件**
   - 打开 `data/print_layouts.json`
   - 找到要配置权限的流程
   - 添加 `permissions` 字段

3. **示例配置**
   ```json
   {
     "PROC-941085FC-98E5-4D84-8144-B81A054C17CB": {
       "name": "设计图纸变更通知单",
       "permissions": {
         "allowed_depts": ["品保部", "采购部", "计划部"]
       },
       "base_pdf": "template.pdf",
       "items": [...]
     }
   }
   ```

4. **保存并测试**
   - 保存配置文件
   - 刷新打印门户页面
   - 验证权限是否生效

### 测试权限配置

使用不同部门的用户账号登录钉钉，访问打印门户，验证：
- ✅ 有权限的用户能看到对应流程的审批单
- ✅ 无权限的用户看不到受限流程的审批单
- ✅ 流程模板筛选器只显示有权限的流程

## 注意事项

1. **部门名称匹配**
   - 必须与 `users.json` 中的 `dept_name` 完全匹配
   - 区分大小写
   - 建议复制粘贴，避免输入错误

2. **用户ID匹配**
   - 必须与 `users.json` 中的 `userid` 完全匹配
   - 使用完整的用户ID字符串

3. **用户信息同步**
   - 定期访问 `/admin/users` 同步用户信息
   - 确保 `users.json` 数据是最新的
   - 新员工入职后需要同步才能正确应用权限

4. **权限生效范围**
   - 权限只控制打印门户的流程列表显示
   - 不影响直接访问打印页面的URL
   - 如需完整权限控制，建议在打印页面也添加权限检查

5. **配置文件备份**
   - 修改 `print_layouts.json` 前建议备份
   - 配置错误可能导致用户无法访问流程
   - 可以通过删除 `permissions` 字段恢复开放访问
